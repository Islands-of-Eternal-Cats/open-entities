# BaseMoveSpeed на каждом юните

Компонент `BaseMoveSpeed` на сущности дублирует значение из YAML для подвижных типов. Это осознанный задел:

**Индивидуальные отклонения от типа** — баффы, усталость, апгрейды и любые эффекты, меняющие скорость у конкретного экземпляра, всё равно потребуют числа «на этом экземпляре». Его логично держать как поле в компоненте ECS, а не только в таблице определений типов.

Альтернатива (читать только из структуры, собранной из YAML) экономит дублирование, пока все юниты одного типа всегда совпадают с дефолтом; как только появляются пер-экземплярные модификаторы, без локального хранения не обойтись или придётся усложнять модель (отдельные компоненты-оверрайды и слияние с типом в системах).
