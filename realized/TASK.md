# Задание: Загрузка сущностей из YAML в Bevy ECS

## Цель

Реализовать загрузку описаний сущностей из YAML-файлов и создание соответствующих сущностей в Bevy ECS по имени типа.

## Контекст

- В проекте уже используется Bevy ECS с компонентами `Position`, `Velocity` и системами движения/вывода.
- Описания сущностей хранятся в файлах и содержат параметры (позиция, скорость и др.).

## Требования

### Обязательные

1. **Формат описаний**  
   Файлы описания сущностей — в формате **YAML**.

2. **Создание сущностей по имени типа**  
   По имени типа сущности из описания в файле создавать соответствующую сущность в ECS (спавн с нужными компонентами).

3. **Интеграция с Bevy ECS**  
   Использовать существующий Bevy ECS: спавн через `Commands`/`World`, компоненты — те же или расширенные (Position, Velocity и т.д.).

### Желательные (по возможности)

4. **Динамические компоненты**  
   Предпочтительный вариант: компоненты определяются по описанию (динамические компоненты).  
   Если в Bevy ECS это неудобно или невозможно — задать набор компонентов заранее и маппить поля YAML на них.

5. **Разный состав компонентов**  
   У разных типов сущностей может быть разный набор компонентов (одни — Position + Velocity, другие — только Position и т.д.). Загрузчик должен это учитывать.

## Ожидаемый результат

- Парсер YAML с описаниями типов сущностей и их параметрами.
- Функция/система: по имени типа из описания создаёт сущность в ECS с нужными компонентами и значениями (позиция, скорость и др.).
- Документация или пример YAML-файла с описанием сущностей.
- При необходимости — обновление AGENTS.md с новыми командами и структурой файлов.

## Ограничения и допущения

- Структуру YAML можно выбрать самостоятельно (например: список типов с полями `name`, `components` или отдельные файлы на тип).
- Если динамические компоненты в Bevy не подходят — допустимо фиксированное множество компонентов (Position, Velocity, и т.д.) с маппингом полей из YAML.
